# PRD — Time Boxing App (가칭: DeepBox)

작성일: 2026-07-09 | 버전: v0.1 (초안)

## 1. 배경 & 문제 정의

<딥 워크>를 읽고 타임박싱을 실천 중이나, 현재 도구(구글 캘린더)에는 두 가지 근본적인 불편이 있다.

1. **입력 마찰이 크다** — 반복되는 Task임에도 매번 텍스트 입력, 시간 설정, 색상 설정을 해야 한다. 기록 행위 자체가 귀찮아지면 타임박싱 습관이 무너진다.
2. **"계획한 시간"과 "실제 몰입한 시간"을 구분하지 못한다** — 캘린더는 2시간짜리 블록을 보여줄 뿐, 그중 실제로 몰입한 시간이 얼마인지 측정할 수 없다. 딥 워크의 핵심인 몰입 밀도를 알 수 없다.

## 2. 목표

- 타임박스 입력을 **3클릭 이내**로 줄인다 (기존 Task 재사용 시 1~2클릭).
- 몰입 세션의 **실제 몰입 시간**(일시정지 제외)을 정확히 기록한다.
- 주간/월간 단위로 **몰입 밀도**(계획 대비 실제 몰입 비율)를 자각할 수 있게 한다.

### 성공 지표 (개인용 앱 기준)
- 하루 타임박싱 기록을 앱에서 완결 (구글 캘린더로 돌아가지 않음)
- 매주 통계 화면을 열어보게 되는가

## 3. 사용자

- 1차: 본인 (딥 워크 실천자, 데스크톱 + 모바일 브라우저 병행 사용)
- 확장 가능성은 열어두되 MVP는 개인용으로 설계

## 4. 기능 요구사항

### P0 — MVP (필수)

#### 4.1 Task 라이브러리
- Task를 미리 등록: 이름, 색상, 카테고리(예: 딥워크/얕은일/휴식), 기본 소요시간
- 자주 쓰는 Task는 상단 노출 (최근 사용순 정렬)
- Task 수정/보관(archive)

#### 4.2 빠른 타임박스 입력
- 오늘의 타임라인(세로형 캘린더 뷰)에서 빈 시간대 클릭 → Task 선택 → 완료 (2클릭)
- 드래그로 시간 범위 지정 가능
- 타임박스 이동/리사이즈/삭제
- 새 Task도 그 자리에서 즉석 생성 가능

#### 4.3 몰입 타이머 (핵심 차별점)
- 타임박스에서 "시작" → 몰입 타이머 가동
- **일시정지 / 재개 / 종료** — 실제 몰입한 시간만 누적 기록
- 비주얼 타이머: 남은 시간이 줄어드는 원형(파이) 시각화
- 타이머 진행 중 화면 상단에 항상 표시되는 미니 타이머 바 (라이브 액티비티의 웹 버전)
- 세션 종료 시 기록: 계획 시간 vs 실제 몰입 시간, 일시정지 횟수
- 브라우저 탭을 닫아도 타이머 상태 유지 (재접속 시 복원)

#### 4.4 통계 & 인사이트
- **주간 뷰**: 총 몰입 시간, 카테고리별 분포, 일별 몰입 시간 추이
- **월간 뷰**: 주 단위 몰입 시간 추이, 최고 기록
- **몰입 밀도**: 계획한 시간 대비 실제 몰입한 시간 비율 (%)
- 계획 없이 흘려보낸 시간대 시각화 (타임박스가 없던 시간)

### P1 — MVP 이후 빠르게

- PWA 설치 지원 (홈 화면 추가 → 위젯처럼 원탭 진입)
- 타이머 종료/휴식 알림 (Web Notification)
- 데이터 내보내기 (CSV/JSON)
- 반복 타임박스 (매일 아침 딥워크 블록 등)

### P2 — 부가 기능

- **AI 타임박싱 제안**: 지난 통계를 기반으로 내일의 타임박스 초안 생성 → 확정 / 수정 후 확정 / 버리기
- AI 주간 회고: 시간 사용 패턴 요약 + 개선 제안

### 명시적 제외 (Out of Scope)
- iOS 네이티브 위젯, 라이브 액티비티 (웹 기술로 불가 — PWA + 미니 타이머 바로 대체)
- 구글 캘린더 양방향 동기화 (v1 제외, 추후 검토)
- 팀/공유 기능

## 5. 화면 구성

| 화면 | 내용 |
|---|---|
| 오늘 (메인) | 세로 타임라인 + 타임박스, 하단 자주 쓰는 Task 퀵버튼, 진행 중 타이머 바 |
| 타이머 | 전체화면 비주얼 타이머 (원형), 일시정지/재개/종료 |
| 통계 | 주간/월간 탭, 몰입 시간·밀도·카테고리 차트 |
| Task 관리 | Task 목록, 추가/수정/보관 |

## 6. 데이터 모델 (초안)

```
Task        : id, name, color, category, defaultDuration, archived, lastUsedAt
TimeBox     : id, taskId, date, startTime, endTime (계획)
FocusSession: id, timeBoxId, startedAt, endedAt, focusedSeconds (실제 몰입),
              pauseCount, pauses[{pausedAt, resumedAt}]
```

- 몰입 밀도 = focusedSeconds / 계획 duration
- 통계는 위 데이터에서 파생 (별도 저장 불필요)

## 7. 비기능 요구사항

- 반응형: 데스크톱 우선, 모바일 브라우저에서도 타이머·기록 가능
- 타이머 정확성: 백그라운드 탭에서도 시간 왜곡 없음 (timestamp 기반 계산, setInterval 의존 금지)
- 오프라인: 최소한 타이머는 네트워크 없이 동작
- 다크 모드 지원
